Introduction:

This module is used to control TXV, a separate piece of software which is a modular video system for generating real-time 2D & 3D visuals. TXV opens in a separate window.

(See TXV General Introduction for more background information.)

Note: The TXV software should be started before trying to connect to the TXV System module.


Controls:

Start TXV - This button will only appear if you are using a TX Modular build which has the TXV app enclosed within it. Once pressed, it will start TXV, and then try to connect to it.
If the button doesn't appear, the user will need to start the TXV app separately.

Connect to TXV - This button will try to connect to TXV. When creating a new TXV system, TXV should be started before connecting to it using this button.

Rebuild TXV System - This button will try to connect to TXV, clear the current system and then rebuild it. This is needed when opening a saved TXV system, or if TXV is restarted for some reason. Make sure TXV is running before rebuilding.

Status - This shows the current status of the TXV system. Initially it will read "Not connected". Once a connection is made, it will read "TXV System connected". When rebuilding a system, it will first read "TXV System rebuild started", then "TXV System rebuild completed" when finished.

Kill TXV - This will stop and close TXV (if it is connected).


TXV System Button

freezeSystem - switch with modulation (def: 0). When freeze is on, all clocks and image updates will be paused.

oscLatency - float (def: 0, range: 0 : 30). When sending OSC messages from the TX Modular to TXV, this adds a delay before executing the message. A small delay (e.g. 0.2 secs) might be used for example, when using a Multitrack Sequencer in the TX Modular to automate TXV parameters - it may help to reduce any jitter in the automation. A longer delay might be needed when running the TXV on a separate computer, where network jitter may be more of a problem.

showFullScreen - switch (def: 0).

windowSizeMode - index (def: 0, range: 0 : 6) options: [0]_Custom Window Size, [1]_480 x 360, [2]_858 x 480, [3]_1280 x 720, [4]_1920 x 1080, [5]_2560 x 1440, [6]_3840 x 2160.

customWindowWidth - integer (def: 1024, range: 1 : 5000).

customWindowHeight - integer (def: 768, range: 1 : 5000).

antiAliasing - switch (def: 1).

neverDropFrames - switch (def: 1). If turned on, every image frame will be created, even if the last frame took longer then time it is allocated based on the targetFrameRate. This is useful when recording video, to make sure frames are never dropped in the recorded video, even if that slows down the movements as displayed in real time.
If turned off, frames will be dropped to make sure that movement never slows down. This is useful for live video where movement - created for example using LFOs and sequencers - needs to appear continuous.

showFrameRate - switch (def: 0). Frame rate is displayed in top left of window. Useful for testing.

targetFrameRate - integer (def: 30, range: 1 : 100).

resetClocks - switch with modulation (def: 0). Resets clocks in all time-based TXV modules - for example LFOs & Step Sequencers, so they restart their sequence or pattern from the beginning.

masterBpm - float with modulation (def: 120, range: 1 : 1000). Any time-based TXV modules, such as LFOs & Step Sequencers, can be set to follow the Master Bpm. This is useful for syncing sequences together and to be able to change speeds in multiple modules at the same time.


Messages Button

This shows messages received from the TXV software.


Draw & Background Button

drawActive - switch with modulation (def: 1).

drawLayersRule - index with modulation (def: 0, range: 0 : 26) options: [0]_All layers except Layer 20, [1]_Layer 1, [2]_Layer 2, [3]_Layer 3, [4]_Layer 4, [5]_Layer 5, [6]_Layer 6, [7]_Layer 7, [8]_Layer 8, [9]_Layer 9, [10]_Layer 10, [11]_Layer 11, [12]_Layer 12, [13]_Layer 13, [14]_Layer 14, [15]_Layer 15, [16]_Layer 16, [17]_Layer 17, [18]_Layer 18, [19]_Layer 19, [20]_Layer 20, [21]_Layers 1 - 5, [22]_Layers 6 - 10, [23]_Layers 11 - 15, [24]_Layers 16 - 20, [25]_Layers 1 - 10, [26]_Layers 11 - 20.

showBackground - switch with modulation (def: 1).

backgroundHue - float with modulation (def: 0, range: 0 : 1).

backgroundSaturation - float with modulation (def: 0.5, range: 0 : 1).

backgroundBrightness - float with modulation (def: 0, range: 0 : 1).

backgroundAlpha - float with modulation (def: 1, range: 0 : 1).

backgroundHSBAsRGB - switch with modulation (def: 0).

alphaThreshold - float (def: 0, range: 0 : 0.05).


TXV Modules Button

Add, show & edit TXV Modules.
New modules are displayed in groups: All Modules, Draw Sources, Image FX, Draw Destinations & Modulation Sources.


Modulation Nodes Button

Add, show & edit modulation Nodes which each connect a modulation source to a module parameter destination.
Also see help file TXV Using Modulation


Draw Connections Button

Add, show & edit Draw Connections which each connect a draw source to a draw destination.
Also see help file TXV Drawing In Layers


Image Connections Button

Show all modules which can import an image directly from another module. Each image import parameter is shown on a separate line. The parameter name and the module to import the image from are shown.


Assets Button

Show, add & remove Assets representing files of the following types: image, movie, svg, cube, 3D model, font.


OSC Address Button

The network address for TXV where OSC messages are sent. This should usually be 127.0.0.1 meaning TXV is running on the same computer as TX Modular.
