TXV Ex 4 Image FX


See also: TXV System, TXV Rendering Images & TXV Assets


Introduction

This help file uses an example file “Ex_4 Image FX" found in the folder "TXV_Examples".

Please open the example in the TX Modular system.

In order to start the TXV software, you can use the “Start TXV” button if you are using the standalone TX Modular (for MacOS) which includes TXV inside it. If you are using another platform, you will need to have a TXV build ready for your platform and open it separately.

Once TXV has been started, press the button “Rebuild TXV System” which will send all the data from the TX Modular to TXV.

You should then see a large red hexagon with a black border rotating in a circle, with multi-coloured stars rising and falling, with the whole image being distorted as it moves.


Overview

This example has been built in stages:

First, the initial image was created by drawing moving shapes to a TXV Render Image module.

Then this image was sent through a chain of Image FX modules to change its colours, add blurring, and finally to add “wobble” which distorts the image.

If you move the sliders on the Interface window, you can see how each of the Image FX changes the image in different ways.


Drawing and rendering the initial image

If you click on the button “Modules + Channels”, you can see in the left-hand column all the modules used.

The red hexagon with a black border is created in TXV Shape 2D [1]. If you open it up and click on the “Shape” button, you will see the shapeType is “Circle”. However, below that the “circleResolution” is set to 6 - in other words the hexagon is a circle made up of 6 lines. If you set the circleResolution to a higher value it will look more like a circle (although the wobble is distorting its shape).


The multi-coloured stars are created in TXV Shape2D Multi [1]. If you open it up and click on the “Shape” button, you will see the shapeType is “Star”.

Below that “numCopies” is set to 16 so there will be 16 stars.

Further down you will see that “circleResolution” is set to 8 giving stars with 8 points.

If you click on the “Colors” button, and click on "Fill Color", you will see the colorHue and colorHueEnd are set to different values. Then click on the “Color Spread” button and you will see the colorSpreadCurve is a linear ramp, meaning that the stars are each given a hue spread evenly between colorHue and colorHueEnd values.


If you open TXV DrawConnection [1] and [2], you can see that the hexagon and stars are drawn to Layers 1 and 2 of the Render Image [1] module, so the stars will be in front of the hexagon.


If you open the TXV Render Image [1] module and click on the “Clear & Render” button and then below that the “Clear” button, you will see that “clearBeforeRender” in on. This means that a background colour, in this case light blue, is drawn on the image before the shapes are drawn.


Chaining Image FX

If you open the TXV Color Controls [1] module and click on the “Image” button, you will see “extSourceImageModule” is set to “TXV Render Image [1]” and “useExternalSourceImage” is set on. This means that this module takes the image created by TXV Render Image [1] and modifies it.

If you click on the “Color Adjust” button, you can see all the ways that the colors on the image can be changed. Try changing some of the values and see what effect it has on the image.


If you open the TXV Blur [1] module and click on the “Image & Blur” button, you will see “extSourceImageModule” is set to “TXV Color Controls [1]” and “useExternalSourceImage” is set on.

This means that the Blur is being added after the Color Controls.


If you open the TXV Wobble [1] module and click on the “Source Image & Wobble” button, you will see “extSourceImageModule” is set to “TXV Blur [1]” and “useExternalSourceImage” is set on.

So the Wobble is being added after the Blur.


Since the Wobble is the last of the Image FX we want to add, the Wobble can then be drawn to the main window.

This can be seen by opening “TXV DrawConnection [3].


If you open “TXV System [1]” and click on the “Image Connections” button, you can see an overview of how the images are chained in the Image FX modules.


Using your own images in the FX chain

First of all you need to add Image Assets to the TXV System:

Open “TXV System [1]”, click on the “Assets” button, and make sure “Asset type” is set to Image.

Now you can drag multiple image files to the yellow box to add them to the system. Alternatively, you can type in the file path of a single image file and use the “Add Image File” button to add images one by one.


Once you have added your image assets, open the TXV Color Controls [1] module and click on the “Image” button.

Turn off “useExternalSourceImage” so that an image asset will be used as the source image instead.

Then you use “sourceImageAsset” to select whichever image you want to use, and your image will now be used in the whole FX chain.

Finally, click on the "Run Interface" button, and move the Image FX sliders to see the how your image is affected.
