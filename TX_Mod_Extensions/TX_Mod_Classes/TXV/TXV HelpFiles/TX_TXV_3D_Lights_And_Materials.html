TXV 3D Lights And Materials


See also: TXV System & TXV 3D Modules


Introduction

Note: The folowing information only applies when you draw 3D shapes to a Scene.

TXV Scene has 3 Lights which each have various colours.

3D Draw Source Modules - such as Shape 3D and Particles 3D - have Material parameters which include various colours as well as Shininess.

The various color compenents of the lights illuminating the scene will interact with the color compenents and shininess of the material each shape is made of, to create the final output image.


Basic Lighting

Note: The following text is quoted from: www.learnopengl.com/Lighting/Basic-Lighting

Lighting in the real world is extremely complicated and depends on way too many factors, something we can't afford to calculate on the limited processing power we have. Lighting in OpenGL [and in TXV] is therefore based on approximations of reality using simplified models that are much easier to process and look relatively similar. These lighting models are based on the physics of light as we understand it. One of those models is called the Phong lighting model. The major building blocks of the Phong model consist of 3 components: ambient, diffuse and specular lighting.

Ambient lighting: even when it is dark there is usually still some light somewhere in the world (the moon, a distant light) so objects are almost never completely dark. To simulate this we use an ambient lighting constant that always gives the object some color.

Diffuse lighting: simulates the directional impact a light object has on an object. This is the most visually significant component of the lighting model. The more a part of an object faces the light source, the brighter it becomes.

Specular lighting: simulates the bright spot of a light that appears on shiny objects. Specular highlights are often more inclined to the color of the light than the color of the object.


Materials

Note: The following text is quoted from: www.learnopengl.com/Lighting/Materials

In the real world, each object reacts differently to light. Steel objects are often shinier than a clay vase for example and a wooden container does not react the same to light as a steel container. Each object also responds differently to specular highlights. Some objects reflect the light without too much scattering resulting in a small highlights and others scatter a lot giving the highlight a larger radius. If we want to simulate several types of objects in OpenGL we have to define material properties specific to each object.

When describing objects we can define a material color for each of the 3 lighting components: ambient, diffuse and specular lighting. By specifying a color for each of the components we have fine-grained control over the color output of the object. Now add a shininess component to those 3 colors and we have all the material properties we need.

The ambient material color defines what color this object reflects under ambient lighting; this is usually the same as the object's color.

The diffuse material color defines the color of the object under diffuse lighting. The diffuse color is (just like ambient lighting) set to the desired object's color.

The specular material color sets the color impact a specular light has on the object (or possibly even reflect an object-specific specular highlight color).

Lastly, the shininess impacts the scattering/radius of the specular highlight.


Emissive Color

Materials have one more color: emissive. This is used for objects that emit light - like a light bulb. The emissive color shines constantly, with no need for a light to illuminate it.


Online Examples Of Material Settings

Please see: http://devernay.free.fr/cours/opengl/materials.html for many examples of material settings.

These include gold, silver, bronze, chrome, jade, pearl, and various types of plastic and rubber.

For each type of material, R/G/B values for Ambient, Diffuse and Specular colors are given, as well as a Shininess value.

Note: The Shininess value needs to be multiplied by 128 to work in TXV. (The website gives shininess values in the range 0-1, but in TXV, it's range is 0-128.).
