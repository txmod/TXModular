OSC Control 2D


Module type: Control Source


Introduction:

This module outputs a control signal from OSC messages received as long as the OSC message begins with a string that matches the OSC string on the screen. The string matching is case sensitive (i.e. upper and lower case letters - G versus g - are treated as different letters, so take care when typing!).

The OSC message sent should consist of the OSC string followed by two numbers. The numbers will be sent out as control signals from this module, mapped to the output range set in the controls.


Note - when sending OSC messages to the TX Modular system, you usually set the port to 57120. (NB: to see which Port the TX Modular System is currently using, go to the Notes & Options screen, see button near the top of the screen.).


Controls:

OSC String - this string will be checked against the first item in any OSC messages received. If the string matches, the control signal will be sent.

OSC Learn - to automatically detect the OSC string - when switched on, this will wait for the next OSC message received and assign the string automatically.

Note: the following OSC strings are used internally by SuperCollider and so shouldn't be used for OSC communication. They will be ignored by OSC Learn:

/status.reply, /tr, /done, /synced, /n_on, /n_off, /n_move, /n_end, /n_go.

Min value - sets the minimum expected value for mapping controller OSC messages to the output range.

Max value - sets the maximum expected value for mapping controller OSC messages to the output range.

Output range - sets the output range of the module.
