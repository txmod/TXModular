OSC Trigger


Module type: Action


Introduction:

This module triggers up to 10 Actions when OSC messages are received as long as the OSC message begins with a string that matches the OSC string on the screen. The string matching is case sensitive - upper and lower case, G versus g, are treated as different letters.

The OSC message sent should consist of the OSC string followed by any arguments, which can be numbers or strings (e.g. 9, 4.77, "on").

OSC message arguments can be passed to the Actions.

For more detials on Actions in the TX Modular system, see help file What_Are_Actions


Note - when sending OSC messages to the TX Modular system, the port usually needs to be set to 57120.


What is OSC?

See help file: OSC_Info


Controls:


OSC String button

OSC String - this string will be checked against the first item in any OSC messages received. If the string matches, the Actions will be triggered.

OSC Learn - to automatically detect the OSC string - when switched on, this will wait for the next OSC message received and assign the string automatically.

Note: the following OSC strings are used internally by SuperCollider and so shouldn't be used for OSC communication. They will be ignored by OSC Learn:

/status.reply, /tr, /done, /synced, /n_on, /n_off, /n_move, /n_end, /n_go.

Triggering - the following choices are given: "Always trigger", "Trigger when next OSC argument is 1", and "Trigger when next OSC argument is 0". The last 2 are useful with on/off button OSC messages recevied for example, from iPad software TouchOSC. These options allow you to run actions specifically when a value is turned ON or OFF - 1 meaning "ON", and 0 meaning "OFF".


Actions 1-4, 5-8, 9-10 buttons

Module 1 - selects which module will be used.

Reset - resets the Action to the default Action (which does nothing).

Action 1 - selects Action will be run out of all the possible Actions for that module.

Settings 1 - depending on which Action is run, values can be set for the Action.

For example, if the Action selected was to set the Frequency of a Waveform module, then a slider would be displayed for setting the frequency.

Use OSC arguments for value settings - if switched on, this means that any arguments that come with the OSC string will be put into the value settings for the Action.

The arguments can either be numbers or text strings.

For example, if the Action selected was to set the frequency of a filter, or the BPM of a sequencer, then a number could be passed to set the value.

If the Action selected had a checkbox to change settings (such as the Reverse checkbox in a Sample Player module), the OSC argument can be: (a) a text string with "ON" or "OFF", or (b) a number with 1 meaning "ON", and 0 meaning "OFF".

If the Action selected had a popup to change settings (such as the type of Waveform in a Wave Synth module), the OSC argument can be: (a) a text string of the popup value, or (b) a number representing the index of the popup. (N.B. Index numbers start from 0, so 0 would be the index of the first item in the popup, 1 would be the index of the second item, 2 would be the index of the third item, etc.)

First argument - this refers to the first argument that will be used for value settings. By default this will be 1, meaning the first OSC argument after the OSC string. Sometimes where OSC messages contain several arguments, higher values may be needed. For example, when using 2D sliders in TouchOSC iPad software, the X and Y values are sent as arguments 1 & 2.
