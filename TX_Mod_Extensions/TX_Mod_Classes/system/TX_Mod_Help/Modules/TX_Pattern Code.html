Pattern Code


Module type: Action


Introduction:

This module allows you to use SuperCollider Pattern code within the TX Modular system, to control any modules in the current system.

Events are used to provide functions to control the system. Each TX event can be controlled using the keys 'dur', 'stretch', "val1", "val2", "val3", "val4", The last 4 are for passing up to 4 values to the Event function (depending on which TX module parameter is being controlled).

A TX Events button (see below) shows all the events available for each module and how many numeric values are required to be set (some require none, others more).


Please Note: It is also possible to control a TX modular system externally using TX events in SuperCollider code:

TXSystem1.arrTXEvents - returns an event containing all available TX events in the currents system.

TXSystem1.displayAllTXEvents - creates a window with all events listed.


Controls in the top row of the module:

Initialise - button to run the initialise code.

Start - button to run the start code.

(NOTE: If the start code is run, the Initialise function will automatically be run first if it hasn't been run at least once since it was compiled.)

Stop - button to run the stop code.

Sync Start - this checkbox sets whether the start code will be run when the "Sync Start" button is pressed. The button is located at the top of the main screen and is used to start multiple modules at the same time.

Sync Stop - this checkbox sets whether the stop code will be run when the "Sync Stop" button is pressed. The button is located at the top of the main screen and is used to stop multiple modules at the same time.


Other Controls:


3 different functions can be edited using buttons:

Edit Initialise Code - a function which initialises any data needed for the pattern to run.

Edit Start Code - a function which starts patterns running and stores any event streams as data.

Edit Stop Code - a function which stops any running event streams.


SuperCollider Code Box - contains the code used by the module. Each function is passed 2 arguments:

arrTXEvents - an event containing events to control all modules in the current system. Use the TX Events button to see all the events available for each module in the system.

dataEvent - an event where any data can be stored and shared between different functions. i.e. data is created in the Initialisation function, and stored in dataEvent. The Start and Stop functions can then access the data and dataEvent is also used to hold the event stream created in the Start function.


Store & compile code - always use this button after you have edited the code, otherwise your edits won't be stored. You will see the result of the compile in the Status box. If the compile isn't successful, relevant error messages will appear on the SuperCollider post window.


Text Insert Tools: Durations, Midi keyboard, Patterns, TX Events - these buttons open banks of extra buttons which will automatically insert code into the code box at the current cursor position.
