Audio Trigger


Module type: Control Source


Introduction:

This module triggers up to 10 Actions when the amplitude (volume level) of a mono audio signal coming into it goes above a certain level (the threshold). Nothing happens when it the amplitude goes from above to below the threshold.

Once an Audio Trigger module has been created, it will appear in the list of possible destinations for audio channels. This allows you to send an audio signal to it from another channel.

It sends out a control signal at the same level as the audio coming in.

For more detials on Actions in the TX Modular system, see help file What_Are_Actions.


Controls:


Trigger button

Trig threshold - The module will only trigger when the incoming audio signal level goes above this threshold. This helps to stop triggering by low level noise.

Reload time - The minimum amount time before a new trigger can happen.


Actions 1-5/ Actions 6-10 buttons

Module 1..10 - selects which module will be used.

Reset - resets the action to the default action (which does nothing).

Action 1..10 - selects action will be run out of all the possible actions for that module.

Settings 1..10 - depending on which action is run, values can be set for the action.

For example, if the action selected was to set the Frequency of a Waveform module, then a slider would be displayed for setting the frequency.

Note: the controls are the same for all other actions.


Audio Input Busses:

Audio in - the incoming mono audio signal.


Control Modulation Input Busses:

Threshold
