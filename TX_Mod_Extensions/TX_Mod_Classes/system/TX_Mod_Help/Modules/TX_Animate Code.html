Animate Code


Module type: Action


Introduction:

This module allows you to use SuperCollider 3 computer code within the TX Modular system, to generate an animation which is displayed in a separate window.

20 modulatable sliders can be used to control the animation.

This module can be used with other modules to create interactive and/or automated audio-visual systems.

NOTE: if the animation seems jerky, try turning off "Show Channel Meters" on the Modules & Channels window & closing the Meter Bridge or Oscilloscope if they're open. You can also lower the "Screen update rate" on the Notes & Options window. All of these changes will give more computer resources to the animation.


Controls:

Controls are on 6 screens accessed by pressing the following buttons:


Window Button

Show Window - this button brings the animation window to the front and reopens it if it is closed.

Close Window - this button closes the animation window.

Window To Front - this button pushes the animation window to the front of all windows.

Window width & height - To specify the size of the display window.

Sample Current W & H - this allows you to resize the window as you want it and then press this button to set Window width & height to the current size.

Default Window W & H - this sets Window width & height to 640 & 480.

Wind. Pos X & Y - To specify the screen position where the window should be displayed.

The X and Y refer to the distance in pixels of the bottom left corner of the window from the bottom left corner of the computer screen.

Sample Current Position - this allows you to move the window to where you want it and then press this button to set Wind. Pos X & Y to the current position.

Default Window Position - this sets Wind. Pos X & Y to near the bottom left corner of the screen.

Border X/ Border Y - To set the width of the black border around the edge of the window, or click on the Presets popup to use a preset border size.

The X and Y refer to the distance in pixels of the left /right and top/bottom borders.

Max FPS - sets the maximum rendering rate (Frames Per Second). If set to 0, it will render as fast as possible. Low FPS values (e.g. 5) will make the playback seem jerky.

Background Colour - for the window with controls for Hue, Saturation & Brightness.

Show Window Controls - turn this off for a borderless window. However, if you want to be able to drag the window around or resize it, then turn it on.

Window Always on Top - keeps the window in front of other windows.

Clear Screen Before Drawing - if on, every time a frame is drawn, the screen will be cleared to the background colour.


Code Button

Enter Supercollider 3 code in the window. The code needs to be a function which uses Pen to draw to the screen. The function will be passed the arguments: animationView - the view itself, frameCount, & arrPassVals - array of 20 modulatable sliders.

Use the Evaulate text button to evaulate the code.


Controls 1-5/ 6-10/ 11-15/ 16-20 Buttons

- The values of each of the 20 sliders are passed to the animation code so that the animation can be controlled live.

Name - name or description for each slider.

Value/Min/Max - these allow you to restrict the range of values of each slider. The Min/Max controls set the minimum and maximum numbers allowed. Value can be modulated.


Control Modulation Input Busses:

Value 1-20
