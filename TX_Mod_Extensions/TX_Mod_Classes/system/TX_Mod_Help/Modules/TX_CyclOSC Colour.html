CyclOSC Colour


Module type: Control Source


Introduction:

This module analyses the video data coming in from a video camera to check for the presence of a particular target colour which you can set.

You can also to define a zone which means that the target is looked for in only certain parts of the video grid. This is useful for triggering events only when a target colour enters a space. For example, by adding multiple modules you can create colour-sensitive hot-zones in an interactive performance or installation space.

Control signals (with values between 0 and 1) are output for the following:

- Out X/ Out Y - When the target colour is found, the module outputs control signals of the X (left to right) and Y (up-down) values of the where the target is found. (If the target is found is more than one grid square, the centre of all the grids where the target is found is used).

(note: The Out X & Out Y control signal outputs can be passed to Smooth insert modules to make their values less "bumpy".)

- Target Found - this control signal outputs a value of 1 if the target is found, otherwise it outputs 0.

When the target colour is found, the module also triggers up to 10 actions. Actions are commands or instructions for any modules in the system.

Examples of actions include: changing the settings of any modules in the system, triggering notes to play, and starting/stopping sequencers.

For more detials on Actions in the TX Modular system, see help file TX_What_Are_Actions.


Note:

This module is designed to be used with the software Processing running on the same computer.

A video camera is also required - a built-in or external camera can be used.

By running a certain file in Processing, it will bring in live camera data and send it to this module to be anaysed (see section Using Processing files below).


Target settings button

Grid size - selects the size of the grid (8x8, 16x16, 32x32) which the video is divided into. The bigger the grid size, the more CPU is used. Note - the grid size must match the grid resolution of the Processing file which you are running.

e.g. Processing file "TXcyclOSC8Col.pde" is for a colour grid of 8 rows and columns.

(see section Using Processing files below).

Take snapshot button - pressing this will fill the video grid with a snapshot of the latest video data that has been sent from Processing. (n.b. If nothing has been sent yet, the grid will become black.)

Video Grid- You can set the target colour by clicking on any square in the video grid (having taken a snapshot first).

Target colour box - this shows what the target colour currently is.

Red - change the red element of the target colour

Green - change the green element of the target colour

Blue - change the blue element of the target colour

Tolerance - ranges from 0-20% - this sets how closely a colour must match the target colour. 0% tolerance means the colours must match exactly. However, colour can easily change due to lighting conditions, so tolerance of around 5% or more is recommended.

Min Target - ranges from 1-20 - this sets the minimum number of grid squares that need to be found to match the target colour.

Analysis rate - ranges from 1-10 - this sets how many times per second the target is searched for.


Zone settings button

Use Zone checkbox - this sets whether the target is searched for in a particular zone (rather than the whole video picture)

Take snapshot button - pressing this will fill the video grid with the latest video data that has been sent from Processing. If nothing has been sent yet, the grid will become black.

Video Grid- You can set which grid squares are in the target zone by clicking on any square in the video grid (having taken a snapshot first). Clicking on any grid square will turn it on or off in the zone. If a grid square is on, it will show its correct colour. If it's off, it will be a pale grey colour.

All On button - this sets all the grid squares on.

All Off button - this sets all the grid squares off.

dragging turns boxes on/ dragging switches box values - when the mouse is clicked and dragged around the grid, this sets whether the boxes it passes are turned on or turned on/off depending their current setting.


Actions 1-4 button

Module 1 - selects which module will be used.

Reset - resets the action to the default action (which does nothing).

Action 1 - selects action will be run out of all the possible actions for that module.

Settings 1 - depending on which action is run, values can be set for the action.

For example, if the action selected was to set the Frequency of a Waveform module, then a slider would be displayed for setting the frequency.

Use action - if switched on, this means that the action will be run.


Action 2/3/4, etc. - the controls are the same for all other actions.


Actions 5-8 button

- the controls are the same for all other actions


Actions 9-10 button

- the controls are the same for all other actions

Using Processing files with this module:


Included Folder: "TX Processing "

There is a folder of Processing files which is included with the TX Modular software.


Running Processing

All the files have been tested with Processing version 1.0.5

In order to use them you will need:

- a recent version of Processing (go to http://www.processing.org/download/)

- the oscP5 library for Processing (go to http://www.sojamo.de/libraries/oscP5/)

Before starting Processing, put the oscP5 folder into the libraries folder of Processing.

Once Processing is running, open a file (see below) and start it running.


Processing files that can be used with this module:

TXcyclOSC8Col.pde

TXcyclOSC8ColNS.pde

TXcyclOSC16Col.pde

TXcyclOSC16ColNS.pde

TXcyclOSC32Col.pde

TXcyclOSC32ColNS.pde


The file names show their grid resolution (8/16/ 32 rows and columns) and colour type (colour or grey).

e.g. File "TXcyclOSC16Col.pde" is for a colour grid of 16 rows and columns.


There are 2 versions of each file. One ends in NS, which means that "no screen" is shown in Processing - this is uses less CPU than the ordinary version.

e.g. File "TXcyclOSC32ColNS.pde" is for a colour grid of 32 rows and columns with no screen shown.


The reason to have 2 versions of each file is to allow you to open the ordinary version of the file first which displays the live video data in Processing. This is to make sure that Processing gets the image from the camera. Once you know this is working, then you can close the ordinary version and open the NS version of the file instead to save CPU load.
