Gametrak


Module type: Control


Note: Only one Gametrak USB controller can connect with the system at a time, so only one Gametrak TX module should be added.


Introduction:

This module is for use with a connected a Gametrak USB controller (originally made for a golf game, but useful as a 2-hand gestural controller with much higher resolution than midi - 10-bit vs. 7-bit).

The controller has 2 strings connected to ball sockets on one end and gloves on the other that go on the user's hands. The position of each hand can be measured at quiet a fine resolution by measuring the rotation angles (X is left-right rotation, Y is front-back) and extension (up to 3 meters) of each string.

All output names begin with "Left" or "Right" which assumes that the user stands in front of the controller with the footswitch on the nearest edge and the USB cable furthest away.

The outputs from each hand include the rotation angle and string extension measurements [Angle X, Angle Y and Extend Z] and the calculated position in 3D space [Position X/Y/Z], with left-right, front-back & up-down as the the X/Y/Z axes.

Note: Position is calculated within a 3-meter high cone-shaped space. So at the bottom tip of the cone, when the string is not extended at all, the Position X & Y values are both 0.5 (centered). As the string is extended, Position X & Y values can move further from the centre, and at the maximum extension they can be anywhere in their full output ranges (0-1).

Range controls adjust the measurement space. For example, the 3 meter extension may not be needed & it may more useful to measure a narrower range.

An easy way to set the measurement ranges is to use the callibration buttons (see below) which allows the user to set all the ranges at once by moving between all the minimum and maximum hand positions in all dimensions. This defines a performance space within which all movements will be measured.

The foot-switch can be used to trigger Actions - it triggers up to 2 Actions each way when it is pressed down or lifted up.

For more detials on Actions in the TX Modular system, see help file What_Are_Actions



Controls:


Reconnect - this tries to reconnect the Gametrak controller if the connection was not found, or after reconnecting the USB cable.

Reset All Outputs - sets all outputs to zero.


Active Ranges button

Calibrate - This will start the calibration process which constantly adjusts the measurement ranges based on the user's movements.

Use Foot Switch to Calibrate - This will start the calibration process which constantly adjusts the measurement ranges based on the user's movements, but ONLY when the foot switch is pressed. This is easier if you need to get into a certain position before calibrating.

Calibrate Off - this stops the calibration process.

Restore Maximum Ranges - resets all measurement ranges to their full range.

Foot-Switch Down Actions/ Foot-Switch Up Actions buttons

Each action has the following:

Module 1 - selects which module will be used.

Reset - resets the Action to the default Action (which does nothing).

Action 1 - selects Action will be run out of all the possible Actions for that module.

Settings 1 - depending on which Action is run, values can be set for the Action.

For example, if the Action selected was to set the Frequency of a Waveform module, then a slider would be displayed for setting the frequency.

The arguments can either be numbers or text strings.

For example, if the Action selected was to set the frequency of a filter, or the BPM of a sequencer, then a number could be passed to set the value.

If the Action selected had a checkbox to change settings (such as the Reverse checkbox in a Sample Player module), the OSC argument can be: (a) a text string with "ON" or "OFF", or (b) a number with 1 meaning "ON", and 0 meaning "OFF".

If the Action selected had a popup to change settings (such as the type of Waveform in a Wave Synth module), the OSC argument can be: (a) a text string of the popup value, or (b) a number representing the index of the popup. (N.B. Index numbers start from 0, so 0 would be the index of the first item in the popup, 1 would be the index of the second item, 2 would be the index of the third item, etc.)
