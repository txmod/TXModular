Action Sequencer


Module type: Action


Introduction:

This is one of the most powerful modules - it can trigger many possible different actions within the overall system at specific times.

Actions are commands or instructions for any modules in the system.

Examples of actions include: changing the settings of any modules in the system, triggering notes to play, starting and stopping other sequencers, as well as causing the Action Sequencer module itself to jump to any step or loop back to the start.

Each step has a probability setting so that uncertainty can be introduced.

For more details about actions, see help files What_Are_Actions and Modulation Vs Automation


Controls:

Start - button to start the sequencer playing from the chosen step.

Stop - button to stop the sequencer.

Sync Start - sets whether the sequencer will be started when the "Sync Start" button is pressed. This button is located at the top of the main screen and is used to start multiple modules at the same time.

Sync Stop - sets whether the sequencer will be stopped when the "Sync Stop" button is pressed. This button is located at the top of the main screen and is used to stop multiple modules at the same time.

Mute all actions - this button is used to mute the sequencer temporarily while it playing. If set on, the sequencer will keep running but no actions will be taken.

Start playing from step - sets which step the sequencer starts playing from.

BPM - sets the tempo in beats per minute.

Tap tempo button - this displays measures the tempo when it is clicked with a mouse on each beat. Use the Command (Apple) key with a mouse drag to copy the BPM to the sequencer BPM.

Auto copy tap tempo - when set on, this automatically copies the tap tempo to the sequencer BPM.

Beats per bar - this affects how bars and beats will be shown on each step.

Keep step times locked when BPM changes - each step shows is start time in minutes and seconds as well as bars and beats. When the BPM is changed, if this button is set on, the time in minutes and seconds for each step will stay the same, and the bars and beats will be recalculated. If this button is off, then the minutes and seconds will be recalculated but the bars and beats will stay the same.


Controls for each step:

[] yellow button to set step highlight on or off, where a yellow box appears behind the row and the action parameters are larger. When a step is edited, it will automatically be highlighted.

add - adds a new step directly after this one with the same step time.

del - deletes this step.

step - sets the step position in the sequence.

ID - each step has a unique ID number which cannot be changed.

mins/secs/bars/beats - sets the step time in both min/sec and bar/beat formats, based on the BPM and Beats per bar settings (see above).

Note - after you change the step time, all steps are sorted to be in time order. This may mean that the step you are editing moves to a new position when you enter its step time.

Also, see the note below: "How to delay the first step in a sequence".

on - if a step is turned off, no action will be triggered.

prob - the probability that an action will be triggered when the sequencer gets to this step. If this is set to less than 100%, then the action will not always be triggered.

upd - if turned on the screen will be updated after this step has run. Useful if an action run in the step changes values that will affect the front interface or another screen.

copy / paste - buttons to copy and past the module/action/value settings between steps.

module - the module that will be instructed to carry out an action.

action - the action to be carried out when the step is triggered. Each module has different actions that can be carried out, so the options given will change depending on the module chosen.

value settings - settings that are relevant to the action: these are given in the name of the action.

For example a Wave Synth module has an action called "Play note: midi note + velocity + gate time". The name indicates that values need to be set for midi note, velocity and gate time.

Some actions don't require any value settings. For example, the Step Sequencer has an action called "Start Sequencer" which doesn't require any value settings.


Notes on using this module:


Getting started

The module is initially created with 2 steps, but you can add as many as you need using the add (a) button.

If you want the action sequencer to carry out a set of actions immediately when started then set all step times to 0 min 0 secs. Otherwise set the time for each step.

If you want to put step times in beats and bars then set the correct BPM and Beats per bar first.

Choose the module before selecting its action.

Use the copy and paste buttons to copy action settings between steps.


Editing the step time (mins/secs/bars/beats)

After you change the step time of any step, all steps are resorted to be in time order. This may mean that the step you are editing moves to a new position when you enter its step time; its step number will also change. (Note - it's step ID will always remain the same).


To move a step, click on the step no and select a new position. The step time (in mins/secs & bars/beats) will change to the same time as the step it is moving to.


Using action "Jump to step ID" for jumping and looping

The Action Sequencer module has an action called "Jump to step ID" that will cause the sequencer to immediately jump to another step. This can be useful for looping a sequence or branching to a different step.

To make the sequence loop to the start when it reaches its last step: add a new step at the end of the sequence with the current Action Sequencer module selected in the "module" box, and "jump to step ID" selected in the "action" box.

Put the step ID of the step 1 into the value settings number box - or use the slider to select the correct ID.

Make sure that step you are jumping back to has an earlier step time than the step your jumping from (see note below).

N.B. Using the "Jump to step ID" action, it is possible that the action sequencer can go into an endless loop. This will only happen if the step you are jumping to is before the current step and has the same step time. If you're looping back, make sure it is to an earlier step time. (If you do this by mistake, an automatic delay of 1/10 sec will be added between each step to stop this crashing your computer).

- See example file: ActionSeqJumpStep for a legit example using the "Jump to step ID" action.


How to delay the first step in a sequence

Whichever step is displayed in "Start playing from step" will be played immediately when the Action Sequencer is started.

This means that if you want the first step to be delayed, you must put an extra step in at a time before it and start the sequencer from the earlier step.

Say, for example, you wanted the current step 1 to start after 3 seconds.

You would need to add a new step and move it to step 1.Then the original step 1 will become step 2.

Then set "Start playing from step" to 1.

Then set step 1's time to 0 min 0 secs and set it to be switched off (so no action will be taken).

Then set step 2's time to 0 min 3 secs.

This would mean that the sequencer starts at step 1 which doesn't trigger any action. Then, after 3 seconds, step 2 runs.
