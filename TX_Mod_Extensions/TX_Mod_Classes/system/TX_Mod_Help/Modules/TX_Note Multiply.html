Note Multiply


Module type: Action


Introduction:

This triggered module makes no sound itself but instead triggers notes in other polyphonic modules. Up to 6 modules can be triggered. It can also output MIDI notes.

When it is triggered, it plays a number of repeated notes. Each note can be tranposed and has its own velocity, probability and gate time settings, allowing randomness to be introduced in various ways.


Controls:

Active - if off then no notes will be triggered


MIDI & Outputs button

MIDI Listen - this allows the module to be triggered by midi Notes, and to respond to midi Pitchbend.

Note: If Midi Listen is turned on make sure none of the individual output modules has their Midi Listen turned on, or there will be double triggering.

MIDI Channels - which midi channels the module will listen to.

Modules - up to 6 polyphonic source modules, such as wave synths, samplers or polyphonic envelopes, can be triggered. All valid modules will appear in the modules list. See the Help Index for details of which modules are polyphonic.

Output MIDI notes for each step - on/off.

MIDI Port - select from available ports

MIDI Loop-Back - sends MIDI notes back into the TX Modular system on the selected channel, so other modules can respond.

Note: Take care not to output MIDI loop-back to a channel that the module itself is listening to - otherwise the system will crash with a MIDI feedback loop.

MIDI Channel - output channel.

MIDI keyboard- this can be played by clicking with a mouse.The midi velocity of the note played is set by the Velocity slider. Middle C is tinted red.


Step Parameters button

Step beats - the length is beats of each step

BPM - beats per minute

Swing - shifts the start of every 2nd step forwards or backwards

Jitter - add random changes to the BPM

Transp. range - the range in semitones of each step’s transpose settings

Quantise transpose to semitones - sets all transpose values to nearest MIDI note

Randomise vel - adds random variation the all velocities.

Velocity scale - if less than 1, notes will have their velocity scaled down.

Scale note velocities to played velocity - this will scale all notes using the played note’s velocity.

Gate range - the range in beats of each step’s gate settings

Gate quantise - if more than 0, this sets the minimum step gate time and quantises it

Ignore transpose for start step - this overrides the step settings. (This might be useful for example, if “Randomise transpose step order” is on, to make sure that the first note is as played.)

Use played velocity for start step - - this overrides the step settings. (This might be useful for example, if “Randomise velocity step order” is on, to make sure that the first note is as played.)

Use probability 100% for start step - - this overrides the step settings. (This might be useful for example, if “Randomise probability step order” is on, to make sure that the first note is always played.)

Randomise transpose step order - this randomly orders the transpose of all the steps.

Randomise velocity step order - this randomly orders the velocity of all the steps.

Randomise probability step order - this randomly orders the probability of all the steps.

Randomise gate beats step order - this randomly orders the gate time of all the steps.


Steps button

Start step - which step to start playing from.

Play steps - the number of steps to play. This will wrap around to the first step if high enough. For example, if the start step is 16 and number of play steps is 4, the steps played will be 16, 1, 2, 3.

Step 1-16 Transpose - transpose semitones for each step

Step 1-16 Velocity - for each step

Step 1-16 Probability - for each step

Step 1-16 Gate beats - for each step
