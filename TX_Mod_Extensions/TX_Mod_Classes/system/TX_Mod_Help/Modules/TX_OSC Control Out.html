OSC Control Out


Module type: Action


Introduction:

This module sends an OSC message to up to 10 different network addresses. The OSC messages have 2 arguments, the OSC string and the control value. The control value can be modulated.

This allows communication across a network to other TX Modular systems running on other computers, or to any software (such as Processing & Max/MSP) that can respond to OSC messages (see help file: OSC_Info)


Controls:


Messsage

OSC String - this OSC string will be sent out.

Latency - the time delay before the message will be sent.

Data rate - the maximum rate of sending OSC messages. The default is 20, but make this 40 or 80 if you are using quickly changing modulation (such as an LFO with a frequency higher than 5 hz).

Value - sets the value to be sent. This value can also be modulated. Use Min-Max to set the range of values that can be sent.

Output type - sets whether Float or Integer values are sent in OSC messages. Note: Integers are created by rounding Float values.

Active - if switched on, OSC messages will be sent out.


Addresses 1-4 / 5-8 / 9-10

Address - the network address (IP Address) of the computer being sent to. (to confirm this, search onlline for "What is my ip address". To check the IP address on an airport network, go to: System Preferences - Network - Airport - Advanced - TCP/IP - IPv4 Address. ).

Port - the port to be sent to. To send OSC messages to another TX Modular system running on the network, you usually set the port to 57120. The default button will set the port to 57120. (NB: to see which Airport IP address and Port the TX Modular System is currently using, go to the Notes & Options screen, see button near the top of the screen.).

Activate - this will activate the address so messages will be sent there.

Notes - a space to put notes - for example the name of the person at the address.
