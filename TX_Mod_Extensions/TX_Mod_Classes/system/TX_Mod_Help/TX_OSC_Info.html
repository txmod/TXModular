What is OSC?

"OpenSound Control ('OSC') is a protocol for communication among computers, sound synthesizers, and other multimedia devices that is optimized for modern networking technology and has been used in many application areas."

[quote from http://www.cnmat.berkeley.edu/OpenSoundControl/].


OSC messages

OSC messages, like MIDI messages, can be used to send note, controller or other data that can be used to interact with other software and other people. This allows for the possibility of networked group instruments where messages flow around the network between different people using different hardware/software. You can communicate with any IP address on the Internet.

- To confirm the IP Address of the computer being sent to, search onlline for "What is my ip address". To check the IP address on an Airport network, go to: System Preferences - Network - Airport - Advanced - TCP/IP - IPv4 Address.

- To see which Port the TX Modular System is currently using, go to the Notes & Options screen, see button near the top of the screen.


OSC in the TX Modular System

OSC messages are sent across a network.

(N.B. On Macs, to create local airport network on one computer which other computers can connect to: click on the Airport icon, make sure it is turned on, then select Create Network.)

To use OSC in the TX Modular system:

- Send OSC messages using the OSC Out and OSC Control Out modules

- Trigger actions from received single OSC messages using the OSC Trigger & OSC Remote modules.

- Receive OSC messages as control signals using the OSC Controller module

(See help files:OSC Controller OSC Control 2D OSC Control Out OSC Out OSC Remote OSC Trigger ).


Note - when sending OSC messages to the TX Modular system, the Port usually needs to be set to 57120. (NB: to see which Port the TX Modular System is currently using, go to the Notes & Options screen). When sending OSC to other software, such as Processing, you need to check what Port is used by that software.


Other OSC software & hardware

There are now lots of pieces of software that can send or receive OSC messages, such as Processing, Reaktor & MAX/MSP/Jitter. Also, some hardware can now send or receive OSC - e.g. iPad, iPhone, Android devices, Monome.
