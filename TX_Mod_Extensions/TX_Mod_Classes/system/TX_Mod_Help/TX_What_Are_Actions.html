What are Actions?

Actions are commands or instructions for any modules in the system.

Various modules can run Actions including: Action Sequencer, Action Slider, Analyser, Audio Trigger, OSC Remote, OSC Trigger. (See their individual help files to see how they use actions).

Also, Action Buttons on the Interface screens can run actions.

Examples of Actions include: changing the settings of any modules in the system such as a frequency or attack time, triggering synth notes to play, and starting or stopping sequencers.

Setting an action has 3 parts:

- the Module that will run the action,

- the Action to be run,

- any Value settings that are used by the action. Some actions don't require any Value settings.

For example:

Action settings: Module: "Step Sequencer [1]", Action: "Set BPM", Value: "145".

Action settings: Module: "Step Sequencer [1]", Action: "Start Sequencer", Value: __.


How to create actions to be run

There are various ways:

- Use an Action Sequencer module to run any number of Actions, either all together or over time (see help file Action Sequencer for more detials).

- Use an Action Button on the Interface window to run up to 10 Actions (see help file 5 Building an Interface for more details).

- Use one of the modules listed above.
