Wave Terrain Builder


Introduction:

The Wave Terrain Builder is an experimental tool to create and visualise wave terrains using maths & code - but without necessarily having to (fully!) understand all the maths or write any code.

Because you can see the terrain produced as a colourful image, you can simply experiment - changing values and seeing what happens to the terrain. There are history buttons to undo/redo any changes.

Once a terrain is built it can be saved as a preset or imported, using drag and drop, into a Wave Terrain module which will turn it into an audio signal - see help file: Wave Terrain


Basic idea:

The Wave Terrain Builder uses up to 4 layers of mathematical functions to create a terrain (see SuperCollider help files for explanations of any mathematical functions).

A range of X (horizontal position) and Y (vertical position) values are passed as inputs to the functions.

Each layer has a central generative function - e.g. sin(x) - which can use numbers as well as X and Y position values to generate terrain values.

After the generative function, up to 5 further 'post' functions can then be applied to the terrain value.
Finally the layer is multiplied by itself (power), a constant number, X & Y position values, and then it is offset.

The layers can be joined in various orders, and using various join functions.

When you have set the parameters and are ready to build, press the Build button, or use Auto-Build to automatically rebuild the terrain whenever any of the parameters is changed.

Note: It can take several seconds to build the Wave Terrain, depending on how complex it is.


Controls:

< > history buttons to go back to previous control settings.

Help - opens the help file

Build Terrain - for manually building. The window colour will become orange during the building process. Once the terrain is built it will be displayed in a separate window.

Auto-Build - if set on, this will automatically rebuild the terrain whenever any of the parameters is changed.

drag & drop - to import/export a wave terrain into a Wave Terrain module

window on top - to keep the window visible, even when it is not selected.

Wave Terrain presets - controls for saving and loading presets. Type in the name box to save a preset - a save or overwrite button will appear when some text has been added. X removes the current text in the box. When you load a preset, it's name appears in the name box so you can overwrite it or save a different version.

Layers Structure - The structure name shows how many and which layers are used, If it has more than 2 layers, the brackets shows the order that layers will be joined - layers in the inner-most brackets will be joined first.

Output range function - if the generated values of a terrain are outside the range -1/+1, they will be kept inside the range using the selected function.

Display - buttons to select which global or layer controls will be displayed below the line (see below).


Global Button:

xMin/xMax/yMin/yMax - set the ranges of X & Y values that will be used for the generated terrain.

Color 1/2/3 HSB - these 3 colors are for terrain values of -1, 0 and 1. Terrain values in between these values will have a proportional mix of nearest colors.

Note: The colors are only used to help with visualisation of the terrain. They make no difference to the sound the terrain will produce when used in a Wave Terrain audio module.


Layer A/ B/ C/ D Buttons:

xOffset/yOffset - offsets the x/y value just for this layer

xInvert/yInvert - inverts the x/y value just for this layer

Layer presets - controls for saving/loading layer presets.

layerJoinFuncIndex - only Layers B, C & D have this control which chooses how the layer will be joined to the preceding layer.

layerGenFuncIndex - >Generative function for the layer which uses the other “gen” controls:

genMult - a multiplier for >the generative function input

genXYFuncIndex - this function controls how X & Y are joined.

genXPower/ genYPower - the power for x/y values

genOffset >- offset to the generative function input

layerPost1FuncIndex…layerPost5FuncIndex - Post functions 1- 5

layerPost1YVal…layerPost5YVal - If the post function is a binary function (e.g. “x + y”), this control sets the y value used in the function.

power, mult, multXPower, multYPower, offset - The layer is multiplied by itself (using power), a number (using mult), X & Y position values (using multXPower & multYPower), and then offset.


Layers Mix Button:

Layers Mix shows only layerJoinFuncIndex, power, mult & offset for each layer in the chosen structure - this is only useful if you are using more than 1 layer.
